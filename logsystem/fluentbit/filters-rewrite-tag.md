# Rewrite Tag

> 创建强大且灵活的路由规则

`Tags` 是创建 `routing` 的关键因素，在配置文件中 Tags 需要在 `[INPUT]` 也就是记录产生的配置段中设置。但是在整个数据采集的 pipeline 中必然有一些场景需要修改 Tags，以便我们创建高级的、灵活的路由规则。

使用 `rewrite_tag` 过滤器可以让记录携带新的 Tags 重新消费[^1]到目标系统。当一条记录被重新消费，这条原始记录可以选择保留或者丢弃。
